FreeBSD 13.0 Updated!

history

Jordan K. Hubbard, Nate Williams and Rodney Grimes are the co-founder of the FreeBSD project. The first FreeBSD release 1.0 appear on November 1st, 1993.

why use FreeBSD

  • UNIX-like
  • Stable and fast general purpose operating system
  • A pretty minimal base system
  • OpenZFS
  • A rescue subsystem
  • Three firewalls in base, PF, IPFW and IPFilter
  • Get bhyve(8), carp(4), dtrace(1), hastd(8), jail(8), linux(4), netgraph(3), netmap(4), rctl(8), vnet(9),
  • A large number of ported software that is up to date
  • Support for multiple architectures
  • Thorough handbook and FAQ
  • Large and friendly community

        # Display the current configuration of network interfaces
        ifconfig

        # Set DHCP for 're0' interface, on the fly
        dhclient re0

        # Perform network (re)initialisation
        sh /etc/rc.d/netif restart

networking (set at startup)

Example 1: configure static IP address for re0


        sysrc ifconfig_re0="inet 192.168.0.58/24"
Don't forget to run sh /etc/rc.d/netif restart to apply changes to running system.

networking (set at startup)

Example 3: configure wireless


        # First, see a list of available wireless networks:
        ifconfig wlan0 scan

      

        File: /etc/wpa_supplicant.conf
        eapol_version=2
        ap_scan=1
        fast_reauth=1

        network={
            ssid="SSID"
            scan_ssid=0
            psk="KEY"
            priority=5
        }
        network={
            priority=0
            key_mgmt=NONE
        }

      

        # Configure wlan
        sysrc wlans_ath0="wlan0"
        sysrc ifconfig_wlan0="WPA DHCP"
Don't forget to run sh /etc/rc.d/netif restart to apply changes to running system.

pf ruleset sample


        File: /etc/pf.conf
        # Protect a laptop (allow only ping/ssh from anywhere)

        set skip on lo
        block log all
        pass in on egress inet proto icmp all icmp-type echoreq
        pass in on egress inet proto tcp from any to any port ssh
        pass out
Read: pf.conf(5)

pf (packet filter)

(Useful commands)


        # Disable PF
        pfctl -d

        # Enable PF and load the rules
        pfctl -ef /etc/pf.conf

        # Just load the rules (apply changes)
        pfctl -f /etc/pf.conf

        # View the loaded rules
        pfctl -s rules
Read: pfctl(8)

openbsd kernels

/bsd
Kernel executable (the operating system loaded into memory at boot-time).

In case of multiple processors, installer will rename /bsd to /bsd.sp and finally /bsd.mp to /bsd

openbsd kernels (continued)

/bsd.booted
Kernel executable, a resume from hibernation (handled by the bootloader).

openbsd kernels

/obsd
Old kernel, next boot it will use the new kernel /bsd (kernel relinking).

openbsd kernels (continued)

/bsd.sp
Kernel executable for single processor machines.

openbsd kernels (continued)

/bsd.mp
Kernel executable for multiprocessor machines.

openbsd kernels (continued)

/bsd.rd

Installation kernel. The built-in RAM disk contains utilities which can be run without an external file system, so this kernel is useful for limited system maintenance too.
